# Healthcare Analytics Dashboard

The Healthcare Analytics Dashboard is the final visualization layer of the Healthcare Lakehouse.

It provides real-time insights into hospital operations, healthcare KPIs, and machine learning predictions using a Databricks SQL Dashboard.

The dashboard reads data directly from the Gold layer and prediction tables generated by the automated workflows.

---

# Purpose

The dashboard is responsible for:

- Visualizing healthcare KPIs
- Monitoring patient admissions
- Tracking readmission trends
- Displaying machine learning predictions
- Supporting clinical and operational decision-making

# Data Sources

The dashboard reads data from the following tables.

| Table | Description |
|------|-------------|
| gold.healthcare_kpis | Business KPI dataset |
| gold.patient_cohorts | Patient cohort analysis |
| gold.encounter_summaries | Encounter summaries |
| gold.readmission_within_30days | Historical readmission statistics |
| gold.predictions | Machine learning prediction results |

# Dashboard Sections

The dashboard consists of two major sections.

## Healthcare KPIs

These visualizations monitor hospital performance.

Examples include:

- Total Patients
- Total Admissions
- Readmission Rate
- Average Length of Stay
- Top Diagnoses
- ADT Volume

---

## Machine Learning Predictions

The dashboard also displays predictions generated by the deployed readmission prediction model.

Prediction insights include:

- Predicted Readmission Risk
- High-Risk Patients
- Prediction Distribution
- Risk Summary

These predictions are refreshed automatically after the inference workflow completes.

# Dashboard File

The project includes the exported Databricks SQL Dashboard.

```
dashboards/

└── Healthcare_Dashboard.lvdash.json
```

This file can be imported into another Databricks workspace to recreate the dashboard.

---

# Importing the Dashboard

To import the dashboard:

1. Open Databricks Workspace.
2. Navigate to **Dashboards**.
3. Click **Import Dashboard**.
4. Select the exported `.lvdash.json` file.
5. Save the dashboard.
6. Update catalog and schema references if necessary.

# Troubleshooting

## Dashboard Shows No Data

Verify that:

- Gold tables contain data.
- Prediction tables contain records.
- The Daily Refresh workflow completed successfully.
- The SQL Warehouse is running.

---

## Predictions Not Visible

Ensure that:

- The inference notebook completed successfully.
- The prediction table has been updated.
- Dashboard queries reference the correct prediction table.

---

## Dashboard Not Updating

Verify that:

- The SQL Warehouse is active.
- Dashboard queries reference the correct catalog and schema.
- The latest pipeline execution completed successfully.
